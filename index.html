Formula1 DataBucket

This project was inspired by my passion for Formula 1. It uses get requests to the Ergast API to retrieve fastest laps/results for tracks and Driver/Team information and the Google Maps API to display the track's location. Unfortunately the API is relatively slow to respond, so the requests sometimes take a few seconds to complete.

An exercise in AJAX, the page only reloads when you click a link in the top navbar. The AJAX requests to the Ergast API take around 300-500 ms to return depending on your internet connection. This caused significant issues when trying to parse through and store the data locally, as the scripts would execute asynchronously and store the same data multiple times. I solved this issue by controlling the flow of the script via callbacks.

The graphs that are displayed on the driver and team pages are created by a custom graphing library that I built from scratch called Charter.js.

Try it out!

VoiceChat

Voice-chat is an app that was incredibly difficult to program and is still a work in progress. It utilizes WebRTC and Socket.io to negotiate a peer-to-peer connection between two broswers and pass their own audio streams to each other. It also has a real-time chat function and user-status updates.

This project utilizes AngularJS for its MVC framework, and a Node.js back-end in conjunction with Express.js, Mongoose, and MongoDB.

EvitOrg

EvitOrg is an event organization app created by Erika and I. It offers the ability to create tasks for an event that you are hosting and assign those tasks to an attendee. Others can also volunteer for a task if they are already attending. It is a Rails app that includes BCrypt, Paperclip, Facebook Login, and Bootstrap.

The database was originally SQLite3 (we changed it to PostgreSQL for deployment) and there is extensive jQuery that was coded by me to make the transitions from page to page look more interesting. In the future, we would like to add custom event-page styling and push notifications when a new event is created by a friend or a task is assigned to you. This would make a great Mobile App!

Image upload is not supported on heroku, as the remote database will not allow images to be uploaded for both space and security reasons.
